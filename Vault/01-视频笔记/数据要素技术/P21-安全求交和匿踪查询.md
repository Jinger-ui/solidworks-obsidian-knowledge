## 详细笔记

> 以下内容基于数据要素流通与隐私计算技术体系撰写，对应 B 站分 P「安全求交和匿踪查询」。**非 UP 逐字转写**；实操步骤请对照视频与官方文档。

### 1. 隐私集合求交 PSI

**PSI**（Private Set Intersection）：两方各持集合 A、B，协议结束后双方仅知 A∩B，不知 A\B 与 B\A 的元素。是多边营销、风控黑名单、联合建模样本对齐的基础。

### 2. PSI 协议族

| 协议 | 特点 |
|------|------|
| 基于 DH/ECC | 高效，适合中等规模 |
| KKRT / OPPRF | 大规模、恶意安全变体 |
| 电路 PSI | 通用但较慢 |
| 多方 PSI | 星型或全连接拓扑 |

### 3. 匿踪查询 PIR

**PIR**（Private Information Retrieval）：客户端从数据库查询一条记录，服务器不知查了哪条。可组合 OT 实现。

**匿踪查询** = PIR + 访问控制：在隐私计算平台中查询对方数据而不暴露查询意图。

### 4. 工程实现要点

- 输入需**规范化**（哈希、加盐、统一格式）
- 规模百万级选 KKRT 类协议
- 恶意安全需额外一致性证明
- 结果仅输出交集 ID，原始属性另走授权通道

### 5. 业务场景

| 场景 | PSI 作用 |
|------|----------|
| 联合营销 | 找共同客户 |
| 联合风控 | 对齐贷款申请人 |
| 广告归因 | 匹配转化用户 |
| 疫情追踪 | 交集风险人员（需合规） |

### 7. 不平衡 PSI

一方集合远大于另一方时用专用协议降低开销。电路 PSI 支持额外谓词（如 age>18）。

### 8. 后续流水线

PSI 输出 → 特征 JOIN → 联邦训练，需在 SecretPad 画布串联组件。

### 9. 法规场景

《反洗钱》要求多方核验不泄露无关客户；PSI+规则 MPC 可在银行间共享可疑主体标签而不暴露全部客户表。
