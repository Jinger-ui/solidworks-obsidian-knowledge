## 详细讲解

### 1. 数联网概念

**数联网**是支撑私域数据在广域范围内安全流通与复用的基础设施网络，类似「数据领域的互联网」。解决孤岛问题，实现**发现、连接、授权、计算、结算**一体化。

### 2. 与数据空间关系

| 概念 | 层次 |
|------|------|
| 可信数据空间 | 单一生态/行业空间 |
| 数联网 | 多空间互联的骨干网络 |
| 数据元件 | 流通的标准化单元 |

### 3. 核心能力

- **统一标识**：数据产品、参与方、合约的全局 ID
- **目录联邦**：跨空间资源发现
- **信任联邦**：身份、证书互认
- **路由调度**：将计算任务路由到最优节点
- **计量结算**：跨空间清分

### 4. 私域数据广域流通

企业数据留在私域（不出域），通过连接器接入数联网，仅暴露**元数据与计算接口**。请求方经授权后远程触发密态计算，实现「数据不动模型动、请求动」。

### 5. 标准与生态

国家数据局推动互联互通标准；运营商、云厂商、行业龙头共建节点。

### 6. 考试/实践要点

- 画数联网三层架构（接入、交换、应用）
- 说明私域与广域流通的技术实现关键
- 对比数联网与传统 API 网关

### 7. DNS 类比

数联网「目录解析」类似 DNS：全局 ID 解析到所在空间与连接器端点。

### 8. 边缘节点

IoT 数据在边缘连接器聚合，仅上传密态统计，减带宽。

### 9. 算力网络

数联网可调度闲置算力节点执行隐私任务，按任务计费，类似「隐私计算的云计算市场」。

## 常见误区

1. **「学完本集就会用隐语」**：SecretFlow 生态需多集串联（P19–P32），单集只是拼图一块。
2. **「隐私计算等于不上传数据」**：数据仍以密文、份额或授权方式参与计算，网络与算力开销客观存在。
3. **「TEE 绝对安全」**：TEE 依赖硬件与侧信道防护，需远程证明（P17）与补丁策略。
4. **「区块链解决一切确权」**：链适合存证与交易撮合，大规模计算仍在链下隐私计算引擎。
