## 核心要点

1. **SolidWorks 正确启动与版本环境**是自动化前提（COM/API 对运行实例敏感）。
2. **自动化准备**包括：插件、宏安全设置、API 引用、Python/C# 运行环境等。
3. 本 P 时长最长，说明环境搭建与排错占比较大——这是后续 API 成功的关键基础。
4. 目标状态：SolidWorks 可被外部程序稳定唤起并保持在可编程状态。

## 详细笔记

### 1. SolidWorks 启动

预期演示内容：

- 正常启动 SolidWorks 并确认许可证/版本
- 新建零件/装配/工程图模板的默认选择
- 与自动化脚本联动的启动方式（先启 SW 再由脚本 attach，或脚本主动创建实例）

### 2. 自动化环境配置

预期涵盖：

| 类别 | 典型项 |
|------|--------|
| SolidWorks 侧 | 宏安全级别、API SDK 帮助文档、引用 SolidWorks.Interop 库 |
| 语言运行时 | Python `pywin32` / C# .NET 项目引用 |
| 路径与权限 | 工作目录、模型保存路径、日志目录 |
| 依赖服务 | 本地 LLM/视觉服务、图纸解析服务（若与 UP 流程一致） |

### 3. 工具链联调准备

- 确认 **COM 连接**：`SldWorks` 应用对象可获取
- 确认 **文档对象**：可创建 `PartDoc` / `AssemblyDoc`
- 准备 **测试用最小图纸/参数**，供 P03 调试
- 记录 **常见报错**（SolidWorks 未启动、RPC 服务器不可用、权限不足等）

### 4. 本 P 学习建议

- 建议边观看边在 `04-代码脚本/` 记录自己的环境清单
- 45 分钟内容建议分两段：① SolidWorks 基础设置 ② 自动化运行时安装验证

## 关键术语

| 术语 | 说明 |
|------|------|
| COM/API | SolidWorks 对外自动化的主要接口方式 |
| SldWorks | SolidWorks 应用程序 COM 对象 |
| 宏安全 | 控制 VBA/宏运行权限的设置 |
| Attach | 脚本连接到已打开的 SolidWorks 实例 |
