## 详细笔记

> 以下内容基于机械 AI + SolidWorks 自动化通用实践撰写，对应 UP 分 P 主题。**非逐字转写**。

### 1. P02 目标

在 P01 明确「建什么」之后，P02 解决「环境怎么跑起来」：启动 SolidWorks、Python/C# 与 COM API 连接、路径与工作目录、模板与单位设置。

### 2. SolidWorks 自动化常见技术栈

| 方式 | 语言 | 特点 |
|------|------|------|
| COM API | Python (win32com)、C#、VBA | 最常用，文档丰富 |
| SolidWorks API 插件 | C# | 深度集成 |
| 宏录制 + 修改 | VBA | 快速原型 |

本系列 likely 使用 **Python + win32com** 或类似方案驱动 SolidWorks。

### 3. 环境检查清单

- [ ] SolidWorks 版本与 API 类型库匹配
- [ ] Python 位数与 SolidWorks 一致（通常 64 位）
- [ ] `pip install pywin32`（若用 Python）
- [ ] 关闭/开启 SolidWorks 实例管理（避免僵尸进程）
- [ ] 工作目录：零件/装配/工程图保存路径
- [ ] 模板：零件 .prtdot、装配 .asmdot 默认模板

### 4. 最小连接示例（概念）

```python
import win32com.client
sw = win32com.client.Dispatch("SldWorks.Application")
sw.Visible = True
model = sw.NewDocument(template_path, 0, 0, 0)
```

实际代码以 UP 演示为准；重点是理解 **Dispatch → NewDocument → Feature 操作** 链路。

### 5. 自动化准备项

**文档管理**
- 新建/打开/保存/关闭的 API 调用顺序
- 文件命名规范（零件号_Rev）

**单位与精度**
- 确保 mm 与图纸一致
- 角度单位、小数位数

**错误处理**
- SolidWorks 未启动时的异常
- 特征失败时的 Rollback 状态

### 6. 与 P01/P03 衔接

P01 输出建模思路 JSON → P02 确保 API 可调用 → P03 将 JSON 字段映射到 API 参数并调试。

### 7. 考试/实践要点

- 能列出自动化环境依赖
- 理解 COM 连接基本原理
- 知道模板路径与 NewDocument 参数含义
- 养成「先手动能建，再写 API」的习惯
