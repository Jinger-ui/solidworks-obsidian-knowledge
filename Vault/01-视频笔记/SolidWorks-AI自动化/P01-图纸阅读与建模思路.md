## 核心要点

1. **二维图纸是 AI 驱动的起点**：整套流程从「读懂机械二维图纸」开始，而非直接三维重建。
2. **先规划建模思路再写代码**：在调用 SolidWorks 之前，需要把图纸信息转化为可执行的建模策略（特征顺序、基准、约束关系）。
3. **AI 的角色是理解与结构化**：将图纸中的尺寸、视图、技术要求抽取为结构化参数，供后续 API 使用。
4. **本 P 为后续自动化的「需求分析」阶段**：P02 起才进入 SolidWorks 环境与 API 落地。

## 详细笔记

### 1. 教程定位

本系列完整版共 6 P，P01 负责建立「从图纸到三维」的认知框架。UP 建议新手按 P01→P06 顺序学习；若只想看结果可先观看精华版。

### 2. 二维图纸阅读（预期内容）

根据分集标题与系列简介，本部分应覆盖：

- **视图识读**：主视图、俯视图、剖视图等如何对应三维形体
- **尺寸与公差**：关键尺寸链、装配关系尺寸的提取
- **技术要求**：材料、热处理、表面粗糙度等对建模/选型的影响
- **零件拆分逻辑**：一张总成图如何拆为多个可独立建模的零件

### 3. 建模思路规划（预期内容）

- 确定 **建模顺序**：基体特征 → 细节特征 → 修饰特征
- 确定 **基准策略**：原点、基准面、基准轴如何与图纸基准对齐
- 确定 **参数化方案**：哪些尺寸应作为驱动参数供 AI/API 修改
- 输出 **建模任务清单**：供 P03 API 流程与 P04 建模脚本消费

### 4. AI 交互边界（预期内容）

- 图纸输入形式（图片/PDF/扫描件）
- AI 输出形式（JSON 参数表、特征列表、自然语言步骤）
- 人工校验点：哪些信息必须人工确认后才能驱动 SolidWorks
