## 详细笔记

> 以下内容基于机械 AI + SolidWorks 自动化通用实践撰写，对应 UP 分 P 主题。**非逐字转写**。

### 1. 为什么从「读图纸」开始

机械 AI 自动化不是「把图片丢给 AI 就出模型」。可靠流程的第一步是**结构化理解二维工程图**：视图关系、尺寸链、公差与技术要求。P01 建立这一认知框架，后续 P02–P06 的 API 与脚本才有明确输入规格。

### 2. 二维图纸的核心要素

**视图体系**
- **主视图**：表达零件最主要外形
- **俯视图 / 左视图**：投影关系（第一角或第三角投影，国内多用第一角）
- **剖视图**：表达内部腔体、孔、筋板
- **详图**：局部放大，标注细小特征

读图顺序：先看标题栏（名称、材料、比例）→ 找主视图 → 对照其他视图建立三维想象 → 提取尺寸。

**尺寸与公差**
- **外形尺寸**：总长、总宽、总高
- **定位尺寸**：孔距、槽位相对基准
- **定形尺寸**：半径、角度、深度
- **公差带**：H7、g6 等配合代号影响选型而非直接建模尺寸
- **形位公差**：⊥、∥、⌖ 等，高级装配与检验用，基础建模可先记录

**技术要求**
- 材料牌号（如 45 钢、铝合金 6061）
- 热处理（调质、淬火 HRC）
- 表面粗糙度 Ra 值
- 去毛刺、倒角 C0.5 等通用要求

### 3. 从图纸到「建模思路」

在写任何 SolidWorks 代码之前，应输出一份**建模任务清单**：

1. **零件拆分**：总成图中有哪些独立零件（螺栓、垫片是否标准件）
2. **基准策略**：原点放哪（通常选对称中心或重要安装面）
3. **特征顺序**：基体（拉伸/旋转）→ 切除 → 孔 → 倒角圆角
4. **参数表**：哪些尺寸作为驱动参数（供 AI/API 修改）
5. **约束关系**：对称、同心、平行等逻辑

**示例思路（假想法兰盘）**
- 基体：中心旋转凸台或拉伸圆盘
- 中心孔：切除 Φd
- 螺栓孔：圆周阵列 n 个 Φd2，节圆直径 D
- 倒角：外缘 C1

### 4. AI 在本流程中的角色

| 阶段 | AI 输入 | AI 输出 | 人工校验 |
|------|---------|---------|----------|
| 读图 | PDF/图片 | 参数 JSON、特征列表 | 关键尺寸、视图对应 |
| 规划 | 参数 JSON | 建模步骤自然语言 | 特征顺序是否合理 |
| 执行 | API 脚本 | 三维模型 | 干涉、配合 |

**边界**：AI 易错在隐藏线、剖面混淆、单位 mm/in 混用；**关键配合尺寸必须人工确认**后再驱动 SolidWorks。

### 5. 常见读图误区

- 把螺纹孔直径当底孔直径（需查标准）
- 忽略「均布」「×4」等阵列标注
- 剖面线区域误判为实体边界
- 对称零件只标一半尺寸，遗漏镜像关系

### 7. 实践建议

1. 选一张简单零件图（单视图 + 剖视），手写参数表
2. 画出特征树草稿（树状：基体→孔→阵列→倒角）
3. 再观看 UP 视频对照其拆分逻辑
4. 勿跳过 P01 直接跑 API——缺结构化输入会导致反复返工
